🚀 PROJECT: THE BUG ANALYTICS ENGINE 
📝 Ngữ cảnh (Context)
Hệ thống của bạn nhận dữ liệu từ một API bên thứ ba. Dữ liệu này thường rất "bẩn" (nested objects, sai kiểu dữ liệu, thiếu trường). 
Bạn cần xây dựng một Engine có khả năng làm sạch dữ liệu, chuyển đổi chúng thành Class Instances và thực hiện đồng bộ hóa với cơ chế Retry chuyên nghiệp.
🛠 Yêu cầu kỹ thuật (Requirements)
1. Custom Error & Validation (OOP)
- Tạo class ValidationError kế thừa từ Error(đây là một built-in).
- Class Issue: Trong constructor, kiểm tra priority. Nếu không thuộc ['High', 'Medium', 'Low'], hãy throw new ValidationError.
2. Advanced Data Transformation (Array & Object)
- Hàm addIssues(rawArray):
- Hãy xử lý các mảng lồng nhau (nested arrays).
- Hãy loại bỏ các bản ghi thiếu id hoặc title.
- Chuyển các raw Object thành Instance của class Bug (nếu type === 'bug') hoặc Issue.
3. Analytics Engine (Reduce Power)
- Hàm getDeepStats(): Chỉ sử dụng duy nhất một lần .reduce() để trả về một Object gồm:
  + total: Tổng số issue.
  + priorityCount: Object thống kê số lượng từng loại priority.
+ byStatus: Group các instance vào mảng Open hoặc Closed.
4. Resilient Sync (Hàm tự giải quyết - Challenges)
- Viết hàm uploadWithRetry(issue, maxRetries). Đây là nơi bạn đối mặt với "Flaky API" (API lúc chạy, lúc lỗi).
